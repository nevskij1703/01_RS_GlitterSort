# Уровни и генератор

## Дефолтные 10 уровней

Хранятся в `Levels.DEFAULT_LEVELS` (массив объектов).

| L | sticks | colors | capacity | empty | difficulty |
|---|---|---|---|---|---|
| 1 | 3 | 2 | 3 | 1 | 4 |
| 2 | 4 | 3 | 3 | 1 | 6 |
| 3 | 5 | 4 | 3 | 1 | 9 |
| 4 | 5 | 4 | 4 | 1 | 12 |
| 5 | 6 | 4 | 4 | 2 | 14 |
| 6 | 7 | 4 | 4 | 3 | 16 |
| 7 | 7 | 4 | 4 | 3 | 18 |
| 8 | 9 | 4 | 4 | 1 | 20 |
| 9 | 10 | 4 | 4 | 2 | 22 |
| 10 | 12 | 4 | 4 | 4 | 24 |

После L10 — endless mode (см. ниже).

## Параметры уровня

| Поле | Тип | Описание |
|---|---|---|
| `sticks` | int 3–13 | Общее число колб |
| `capacity` | int 2–6 | Сколько слоёв влезает в одну колбу |
| `emptyStick` | int ≥0 | Пустых буфер-колб в стартовой раскладке |
| `colours` | string[] | 2–4 имени цвета из `Levels.PALETTE` |
| `difficulty` | int 1–100 | Целевое число ходов (точное для V2, нижняя граница для V1) |
| `generator` | `'reverse-shuffle'` \| `'random-tuned'` | Какой алгоритм использовать |
| `tuningTimeoutMs` | int | Лимит на random-tuned (мс) |
| `easyMode` | bool | Только для V2: гарантировать отсутствие тупиков |
| `easyModeMaxAttempts` | int | Максимум попыток при easyMode (default 100) |
| `distinctLayers` | bool | Эмпирическая ускоряющая эвристика: каждый слой = разные цвета |
| `seed` | int | Зерно PRNG; обеспечивает детерминизм |
| `bottles` | string[][] | Опционально — готовая раскладка (тогда генератор не вызывается) |

## Палитра

Хранится в `Levels.PALETTE` и CSS:

```
pink, orange, yellow, lime, teal, blue, purple, crimson
```

Чтобы добавить цвет: вписать имя в `PALETTE`, добавить CSS-переменную `--c-<name>` в `:root`, и CSS-правило `.bottle .layer.c-<name> { background: var(--c-<name>) }`.

## Два генератора

### V1 — reverse-shuffle (быстрый, всегда решаемый)

Стартует с уже решённого состояния (каждая колба моноцветная) и применяет `difficulty` обратных ходов с фильтрами против самоотмены/повторов. Решаемость гарантирована построением; `difficulty` — это нижняя граница (реальное число ходов может быть чуть выше из-за compact-фазы, восстанавливающей пустые буферы).

**Когда использовать:** очень быстро (миллисекунды), для прототипов или когда точное число ходов не важно.

### V2 — random-tuned (медленный, точный)

Генерирует случайную полную раскладку, прогоняет через BFS-решатель, и выбирает раскладку с `BFS-длина == difficulty`. Если в бюджете `tuningTimeoutMs` точное совпадение не найдено — берём ближайшее.

**Easy Mode** (`easyMode: true`): дополнительно проверяет каждую найденную раскладку на отсутствие тупиков через полный обход графа достижимых состояний (`Solver.findReachableDeadEnds`). Только полностью dead-end-free раскладки идут в `best`.

**Когда использовать:** для production-уровней с балансом сложности. Обязательно для endless mode (там `easyMode: true`).

## Endless mode (L11+)

Любой уровень N > `Levels.DEFAULT_LEVELS.length` процедурный. Генерируется через `generateEndlessConfig(N)`, конфиг кэшируется в `_endlessCache[N]`.

Прогрессия по тирам — таблица `Levels.ENDLESS_TIERS`. Каждый тир задаёт `(sticks, capacity, emptyStick, colors, difficulty)`. Для текущего N берётся тир `(N - 11) mod tier.length`. Цвета — случайные 4 из палитры. Seed случайный — каждый endless-уровень уникален между сессиями (но в рамках одной сессии стабилен через кэш).

Чтобы изменить кривую сложности — править `ENDLESS_TIERS`.

## Дев-панель (`Ctrl+Shift+D`)

Открывает форму прямо поверх игры.

| Поле | Что делает |
|---|---|
| `Колбочки` | sticks |
| `Ёмкость` | capacity |
| `Пустых` | emptyStick |
| `Цвета` | toggle-свотчи, лимит 2–4 |
| `Генератор` | reverse-shuffle / random-tuned |
| `Сложность` | difficulty (целевые ходы) |
| `Тайм-аут` | tuningTimeoutMs |
| `Easy Mode` | easyMode |
| `Distinct layers` | distinctLayers |
| `Seed` | seed PRNG |

**Кнопки:**

- **🎲 Сгенерировать** — применяет параметры к текущему уровню как **runtime override** (в памяти, теряется при reload). Показывает meta генератора в строке статуса: `attempts=N, solvable=N, optimal=N, distance=N ✓, Xms`.
- **💾 Сохранить override** — записывает текущий конфиг в `localStorage.customLevels[N]`. Этот override переживёт reload и будет грузиться вместо дефолтного уровня N.
- **↺ Сбросить override** — удаляет и runtime, и persistent override для текущего уровня. После этого нужно перезагрузить уровень (выйти → войти, или нажать Restart).
- **📋 Copy JSON** — копирует текущий конфиг в буфер обмена. Полезно, когда отбалансированный уровень нужно вкомпилировать в `DEFAULT_LEVELS` источника.

### Чтение статуса генератора

Пример: `attempts=487, solvable=234, optimal=12, distance=0 ✓, 4823ms`

- `attempts` — сколько раскладок сгенерировал V2 за время бюджета.
- `solvable` — сколько из них прошли BFS (имеют решение).
- `optimal` — длина оптимального пути найденной лучшей раскладки.
- `distance` — `|optimal - difficulty|`. `0 ✓` = точное попадание в цель.
- `Xms` — затраченное время.

Если `distance > 0` — увеличь `tuningTimeoutMs` или измени параметры (другой `seed`, больше колб, …).

## Каденс уровней

Для геймплея удобно проверить весь tower в дев-панели:

1. `Ctrl+Shift+D`
2. Поменяй `seed` → 🎲 Сгенерировать → запомни/скриншоть `optimal`
3. Если устраивает — 📋 Copy JSON → перенеси в `DEFAULT_LEVELS` в source code.

Имена цветов в `colours[]` должны быть из `Levels.PALETTE`.
